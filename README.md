📦 Predictive Inventory Forecasting using Machine Learning
📌 Business Problem

Service centers face challenges in maintaining optimal spare parts inventory due to uncertain and intermittent demand. Excess stock increases holding costs, while stockouts impact service quality.

🎯 Project Objective

Develop a predictive forecasting model to help service centers achieve Just-In-Time (JIT) inventory standards.

📊 Dataset Overview

- 28,484 service transactions

- 7 structured features

- 845 days of historical demand

- 80% zero-demand days (intermittent demand)

🛠 Tech Stack

- Python (Pandas, NumPy)

- SQL (Data Extraction)

- XGBoost, LightGBM, Random Forest

- ARIMA, SARIMA, Prophet

- Statistical Tests (ADF, Durbin-Watson)

- Inventory Optimization (EOQ, Safety Stock, ROP)

🔎 Approach

- Data Cleaning & Quality Validation

- Exploratory Data Analysis

- Time Series Preparation

- Feature Engineering (Lag, Rolling, EMA, Seasonal Features)

- Model Comparison (10 Forecasting Models)

- Model Validation & Residual Analysis

- Inventory Optimization Strategy

🏆 Best Model Performance

- Model: XGBoost

- R²: 0.94

- Lowest RMSE

95% Confidence Interval Coverage: 91.7%

📦 Inventory Recommendations

- Safety Stock (95% service level): 1.4 units

- Reorder Point: 2.1 units

- EOQ: 29 units

💡 Business Impact

- 20–30% Inventory Cost Reduction

- Reduced Stockouts

- Improved Service Levels

- Better Working Capital Management
